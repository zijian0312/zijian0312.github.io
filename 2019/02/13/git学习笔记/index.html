git学习笔记

Feb 13, 2019

git 配置

配置个人用户名和email

如果用了--global选项，那么更改的配置文件就在用户主目录下，以后所有的项目都默认使用该配置。

如果要在某个特定的项目中使用其他名字和电邮，只有去掉--global选项重新配置即可，
新的设定保存在当前项目的.git/config文件里。

查看配置信息

git config --list

有时候会看到重复的变量名，那就说明它们来自不同的配置文件（比如 /etc/gitconfig 和 ～/.gitconfig），不过最终 Git 实际采用的是最后一个。

查阅某个环境变量的设定，只要把特定的名字跟在后面即可。如：
git config user.name

git 工作流程

git 命令

git reset HEAD: 暂存区的目录树会被重写，被 master 分支指向的目录树所替换。工作区不受影响。

git rm –cached : 直接从暂存区删除文件，工作区则不做出改变。

git checkout .” 或者 “git checkout –
用暂存区全部或指定的文件替换工作区的文件。
这个操作很危险，会清除工作区中未添加到暂存区的改动。

git checkout HEAD .” 或者 “git checkout HEAD
会用 HEAD 指向的 master 分支中的全部或者部分文件替换暂存区和以及工作区中的文件。
这个命令也是极具危险性的，因为不但会清除工作区中未提交的改动，也会清除暂存区中未提交的改动。

git 创建仓库

在指定目录下创建仓库 git init
以指定目录作为仓库 git init repo

git clone

使用 git clone 拷贝一个 Git 仓库到本地
默认情况下，Git 会按照你提供的 URL 所指示的项目的名称创建你的本地项目目录。
如果你想要一个不一样的名字， 你可以在该命令后加上你想要的名称。

git add file

git add 命令可将该文件添加到缓存。

git add .

添加所有修改文件到缓存。

git status

查看项目的当前状态。使用-s 参数以获得简要结果。

git diff

执行 git diff 来查看执行 git status 的结果的详细信息。
git diff 命令显示已写入缓存与已修改但尚未写入缓存的改动的区别。

  • 尚未缓存的改动： git diff
  • 查看已缓存的改动： git diff ­­cached
  • 查看已缓存的与未缓存的所有改动： git diff HEAD
  • 显示摘要而非整个 diff： git diff ­­stat

git commit

使用 git add 命令将想要快照的内容写入缓存区， 而执行 git commit 将缓存区内容添加到仓库中。
Git 为你的每一个提交都记录你的名字与电子邮箱地址，所以第一步需要配置用户名和邮箱地址。

git commit ­-a或者git commit -am '修改txt文件'，跳过git add提交缓存的过程。

git reset HEAD –file

取消其中一个文件的缓存。

git rm

git rm 会将条目从缓存区中移除。这与 git reset HEAD 将条目取消缓存是有区别的。 取消缓存的意思就是将缓存区恢复为我们做出修改之前的样子。
默认情况下，git rm file 会将文件从缓存区和你的硬盘中（工作目录）删除。
如果你要在工作目录中留着该文件，可以使用 git rm --cached file。

git mv

git mv 有三个操作，

  1. git rm --cached file
  2. 修改文件名。
  3. git add file

git分支管理

几乎每一种版本控制系统都以某种形式支持分支。使用分支意味着你可以从开发主线上分离开来，然后在不影响主线的同时继续工作。

git branch 分支名

创建分支。

git checkout 分支名

切换分支。
当你切换分支的时候，Git 会用该分支的最后提交的快照替换你的工作目录的内容， 所以多个分支不需要多个目录。

git merge 合并分支

你可以多次合并到统一分支， 也可以选择在合并之后直接删除被并入的分支。

git branch

列出分支。


当你以此方式在上次提交更新之后创建了新分支，如果后来又有更新提交， 然后又切换到了”testing”分支，Git 将还原你的工作目录到你创建分支时候的样子。

git checkout -b 分支名

创建分支并切换到该分支。

git branch -d 分支名

删除分支。

git merge 分支名

将指定分支合并到当前分支。

合并冲突

合并后的冲突会在文件中显示，需要手动修改。

git 查看提交历史

git log 列出历史提交记录
git log --oneline 查看历史记录的简洁版本。
git log --oneline --graph 使用拓扑图来查看历史记录。
git log --reverse --oneline 使用--reverse参数逆向显示所有日志。
git log --author=zhangsan --oneline -5 查看指定用户提交的日志， -5表示输出行数。

如果你要指定日期，可以执行几个选项：--since 和 --before，但是你也可以用 --until 和 --after。

例如：查看三周前且在十月十八日之后的提交
git log --oneline --before{3.weeks.ago} --after={2018-10-18} --no-merges
–no-merges选项可以隐藏合并提交。

git　标签

git tag -a v1.0 给最新一次提交打上“v1.0”的标签。
-a选项意为创建一个带注解的标签。不用 -a 选项也可以执行，但是不会记录标签的详细信息。
推荐使用带注解的标签。

git log --decorate 可以看到建立的标签。
git tag -a v0.9 85fc7e7 给某一次提交追加标签。
git tag 查看所有标签
git tag -a 标签名 -m "message" 指定标签信息
git tag -s 标签名 -m "message" PGP签名标签

git tag -d v1.0

删除标签。

git 远程仓库

第一次提交到远程仓库

  1. 在github上新建一个公共或私有仓库，可以先不用创建README文件。
  2. 在本地目录编辑好README文件和要创建的代码文件后，按github上的
    提示完成初始化和第一次提交工作。
  3. 此时，本地文件已上传至github仓库。

git remote 查看当前配置的远程仓库。
git remote -v 可以看到每个别名的实际地址。

提取远程仓库

Git有两个命令用来提取远程仓库的更新。

  1. 从远程仓库下载新分支与数据：

    git fetch，然后git merge 远程分支到所在分支。

  2. 从远程仓库提取数据并合并到当前分支：

    git pull

推送到远程仓库

git push [alias] [branch]
将branch 分支推送成为远程仓库的branch分支。

删除远程仓库

git remote rm [alias]

其他问题

忽略某些文件和文件夹的提交

  1. 命令方法
    在仓库中创建隐藏文件 .gitignore
    添加要忽略的文件或文件夹，如：
    /target/
    *.project
    *.iml

  2. 修改编辑器git插件，添加要过滤的文件。
    该方法只能保证文件提交过滤，远程仓库不会。

删除github上多余的文件夹

  1. git pull origin master
  2. dir
  3. git rm -r --cached target
  4. git commit -m '删除了target'
  5. git push -u origin master
